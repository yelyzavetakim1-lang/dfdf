Methodology and Instrumental Tools for Developing Authentication Systems Based on the Zero Secret Server Authentication Model

A structured presentation of the research topic, methodology, expected outcomes and project materials.

The thesis explores architectural approaches, tools and practices for building authentication systems where critical user secrets are not stored on the server and security-sensitive logic is shifted closer to the client side.

01 · Research

Relevance and research scope

Why it matters

Traditional systems often store user secrets or their derivatives on the server. This increases the attack surface, complicates secure data handling and raises the risk of compromise in the event of database leaks or configuration errors.

What is studied

The thesis examines approaches to authentication where trust is shifted away from server-side secret storage and toward cryptographic mechanisms on the client, while the server mainly coordinates operations without permanently storing sensitive secret material.

02 · Goal and tasks

What the study is designed to achieve

Research goal

To investigate methodological approaches and instrumental tools for developing authentication systems based on the Zero Secret Server Authentication model and prepare the foundation for building a demonstration prototype.

  1. 01

    Analyze existing approaches to authentication system implementation.

  2. 02

    Examine the core principles of the Zero Secret Server Authentication model.

  3. 03

    Identify development tools suitable for a secure web-based solution.

  4. 04

    Define an architectural approach for building the authentication system.

  5. 05

    Prepare the foundation for the future implementation of a demonstration prototype.

03 · Methodology

Methods and implementation perspective

The research combines analysis of modern authentication models, secure web development practices, client-side encryption, a stateless approach to authentication data processing and a comparative review of implementation tools for web systems.

Source analysis

Review of scientific and technical sources to build the theoretical basis of the thesis.

Architecture comparison

Evaluation of strengths and trade-offs of classic server-centric and Zero Secret approaches.

System modeling

Design of the logical structure of the web system and its key components.

Landing page design
